# VRMUtils最適化
「最適化（Optimization）」という言葉はプログラミングでよく出てきますが、ここでは具体的に**「描画の負荷を減らして、動作を軽くする」**ことを指しています。

なぜこの処理が必要なのか、一緒に深掘りしてみましょう。

まず、3Dキャラクターが動く仕組みをイメージしてみてください。キャラクターの中には「骨（ボーン/Joint）」が入っていて、その骨が動くと表面の皮（メッシュ）がついてきます。

しかし、モデリングソフト（BlenderやVRoid Studioなど）から書き出したデータには、実は**「見た目には全く影響しない、空っぽの骨」**がたくさん含まれていることがあるんです。

ここからがポイントなのですが、Three.jsなどの3Dエンジンは、画面を更新するたびに（1秒間に60回など）、**全ての骨の位置や計算**を行おうとします。

もし、その中に「動かしても見た目が変わらない無駄な骨」が大量にあったら、コンピュータはどうなると思いますか？
